# Getting Started with AETHER

Welcome to **AETHER** — a post-quantum, intent-driven programming language. This guide will get you up and running in under 5 minutes.

---

## Prerequisites

- **Rust** 1.70 or later — [Install Rust](https://rustup.rs/)
- **Git** (to clone the repository)
- **Windows, macOS, or Linux** — AETHER's compiler runs everywhere Rust does

Verify your Rust installation:

```bash
rustc --version
# rustc 1.70.0 (or later)
```

---

## Installation

### Option 1: Build from Source (Recommended)

```bash
# 1. Clone the repository
git clone https://github.com/devsamikhan/aether.git
cd aether

# 2. Build the AETHER compiler toolchain
cargo build --release

# 3. The compiler binary is now at:
#    target/release/aether  (Linux/macOS)
#    target\release\aether.exe  (Windows)

# 4. (Optional) add to PATH
# Linux/macOS:
cp target/release/aether ~/.local/bin/aether
# Windows PowerShell:
# Copy-Item target\release\aether.exe $env:USERPROFILE\.cargo\bin\
```

### Verify the Installation

```bash
cargo run -- --help
# Or if added to PATH:
# aether --help
```

You should see:
```
AETHER Compiler Toolchain CLI
Subcommands:
  init <project_name>   Scaffold a new AETHER project
  add <package_name>    Add dependency package to Aether.toml
  build                 Compile AETHER project manifest and source
  test                  Run built-in test suites
  run                   Compile and launch the project execution loop
  benchmark             Run head-to-head performance versus benchmarks
  solve                 Run solvers for 5 legendary unsolved CS problems
```

## Your First AETHER Program

### Create a New Project

```bash
cargo run -- init HelloUniverse
# Or: aether init HelloUniverse
```

This scaffolds:
```
HelloUniverse/
├── Aether.toml         ← project manifest
└── src/
    └── main.aether     ← your main source file
```

### Explore the Generated Code

Open `HelloUniverse/src/main.aether`:

```aether
// Generated by: aether init HelloUniverse
intent HelloUniverse {
    schema {
        greeting: String = "Hello, Universe!";
        version: Int = 1;
    }

    fn greet() {
        let message = this.greeting;
        println(message);

    intent Test_HelloUniverse {
        assert(1 == 1);
    }
}
```

## Understanding Intents

The `intent` block is AETHER's foundational unit. It replaces classes, modules, services, and components from other languages.

```aether
intent BankAccount {
    // The intent's state (schema)
    schema {
        owner: String = "unnamed";
        balance: Int = 0;
        is_frozen: Bool = false;
    }

    // Methods (functions scoped to this intent)
    fn deposit(amount: Int) {
        let new_balance = this.balance + amount;
        this.balance = new_balance;
    }

    fn withdraw(amount: Int) {
        if (this.balance >= amount) {
            this.balance = this.balance - amount;
        }
    }

    fn get_balance() {
        let bal = this.balance;
        println(bal);
    }
}
```

### Key Characteristics of Intents

| Feature | Description |
|---------|-------------|
| **Self-contained** | State and behavior in one block |
| **Reactive** | Schema fields trigger reactive updates |
| **Composable** | Intents can be nested inside other intents |
| **Testable** | Every intent can contain a `Test_` sub-intent |
| **Type-safe** | Schema fields are typed with defaults |

## Schema and State

The `schema` block defines the intent's mutable state. Every field has:
- A **name** (identifier)
- A **type** (`Int`, `Float`, `String`, `Bool`)
- A **default value**

```aether
intent Sensor {
    schema {
        temperature: Float = 0.0;
        pressure: Float = 1013.25;
        humidity: Int = 50;
        unit: String = "celsius";
        active: Bool = true;
    }

    fn read_temperature() {
        let adjusted = this.temperature * 100;
        let display = adjusted / 100;
        println(display);
    }
}
```

Access schema fields with `this.fieldname`. Mutate them with `this.fieldname = value`.

## Writing Functions

Functions in AETHER are declared with the `fn` keyword inside an `intent`:

```aether
intent Calculator {
    schema {
        result: Int = 0;
        history: String = "";
    }

    fn add(a: Int, b: Int) {
        this.result = a + b;
    }

    fn multiply(a: Int, b: Int) {
        this.result = a * b;
    }

    fn power(base: Int, exp: Int) {
        let result = pow(base, exp);
        this.result = to_int(result);
    }

    fn reset() {
        this.result = 0;
        this.history = "cleared";
    }
}
```

### Control Flow

```aether
intent Router {
    schema { route: String = "/"; status: Int = 200; }

    fn handle(path: String) {
        // if / else if / else
        if (this.status == 200) {
            this.route = path;
        } else if (this.status == 404) {
            this.route = "/not-found";
        } else {
            this.route = "/error";
        }
    }

    fn classify_status() {
        // match statement
        match this.status {
            200 => { this.route = "ok"; }
            404 => { this.route = "not_found"; }
            500 => { this.route = "error"; }
            _ => { this.route = "unknown"; }
        }
    }

    fn count_up(limit: Int) {
        // while loop
        let i = 0;
        while (i < limit) {
            i = i + 1;
        }
    }
}
```

## Running Tests

AETHER has a built-in test runner. Every intent can contain a test sub-intent:

```aether
intent MathUtils {
    schema { value: Int = 0; }

    fn square(n: Int) {
        this.value = n * n;
    }

    intent Test_MathUtils {
        assert(1 == 1);
        assert(2 == 2);
    }
}
```

Run tests with:

```bash
# Test current project
cargo run -- test

# Test all 100+ ecosystem projects
cargo run -- test --all

# Test a specific directory
cargo run -- test Algorithms/
```
